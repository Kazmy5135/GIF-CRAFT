# 序列帧工作区 MVP 开发计划

## 目标

- 接收 `GENERATION` 已完整交接并已在本地持久化的稳定帧集合，不绕过现有完整性守卫。
- 让用户按任务帧率和循环方式连续预览动画，并能逐帧查看原图、序号、尺寸和审核状态。
- 支持帧筛选、非破坏性移除与恢复、稳定排序，以及指定帧重试入口。
- 保留原始生成任务和原始帧不变，把工作区编辑保存为独立、可恢复、可追踪的工作副本。
- 输出一个顺序稳定、资源可读、元数据完整的工作区快照，作为后续 `EXPORT` 模块的唯一输入。
- 在刷新、页签切换和重新进入任务后恢复工作区，且不会因本地清理误删仍被工作区引用的资源。

## 非目标

- 不在本模块实现 ZIP、GIF、WebP 或视频编码与下载。
- 不提供绘画、擦除、蒙版、裁剪、调色、图层、补间、洋葱皮或专业时间轴编辑。
- 不支持跨任务拼接、多项目素材库、团队协作、云端同步或跨设备恢复。
- 不自动执行 AI 连续性评分、异常帧检测或首尾循环质量判定。
- 不在 MVP 中提供可变播放速度、逐帧时长、音频轨道或关键帧曲线。
- 不直接访问具体 AI 服务商；所有重试必须经过 `GENERATION` / `AI_GATEWAY` 的公开用例。
- 不把“删除”实现为立即删除原始 Blob；原生成结果在其保留期内保持不可变和可追溯。

## 判级理由

本变更新增完整的帧审核与编辑用户流程，定义工作区聚合、持久化结构、重试状态和导出交接契约，并改变 `APP`、`CORE`、`STORAGE`、`GENERATION` 与未来 `EXPORT` 的公开协作边界，属于大模块。

## 影响模块

- `FRAME_WORKSPACE`：新增页面、状态容器、工作区用例、预览、审核、排序、移除、恢复和重试交互。
- `CORE`：新增工作区聚合、帧决策、排序规则、重试尝试和导出快照契约；原 `Frame` 继续保持不可变。
- `STORAGE`：增加工作区元数据、替换候选资源和迁移；清理策略必须保护工作区引用。
- `GENERATION`：提供按 `jobId` 读取完整交接的入口，并承接能力驱动的指定帧重试任务。
- `AI_GATEWAY`：公开 `frameRetryMode` 能力，并以当前代理实例查询已持久化的重试子任务。
- `APP`：将 `/frames` 占位页替换为真实页面，处理带或不带 `jobId` 的进入路径。
- `EXPORT`：本轮只冻结其消费的 `FrameWorkspaceSnapshot` 契约，不实现编码或下载。

## 当前行为

- `/frames` 仍是占位页，不能读取 `jobId`、本地任务或帧 Blob。
- 生成页只在任务状态为 `completed`、结果完整性为 `complete`、本地资源状态为 `available` 且帧数大于零时显示“进入序列帧工作区”。
- 现有交接契约已提供 `jobId`、预设、按 `sequenceIndex` 排序的 `Frame[]`、帧率、循环方式、画布和锚点。
- `Frame.providerIndex` 与 `Frame.sequenceIndex` 是不可变来源信息；当前领域已明确后续展示顺序由帧工作区拥有。
- IndexedDB v2 的 `generation-jobs` 与 `frame-resources` 可恢复完整任务和帧 Blob，但没有工作区编辑状态、替换候选或工作区资源保护关系。
- 当前 Gorilla MCP Seedance 2.0 通过完整视频生成并抽取 8/12 帧，不提供原生单帧任务、外部任务 ID、查询、取消或真实百分比。

## 目标行为

### 主流程

```text
完整 GenerationJob + 可读 Frame[]
  → 通过 jobId 创建或恢复 FrameWorkspace
  → 按任务 FPS / loopMode 连续预览
  → 逐帧审核（待审核 / 保留 / 已移除）
  → 拖拽或按钮稳定排序
  → 对问题帧发起重试并预览候选
  → 明确接受候选或放弃候选
  → 校验工作区可用性
  → 生成不可变 FrameWorkspaceSnapshot
  → 后续 EXPORT 消费
```

### 进入与恢复

- 从生成页进入时使用 `/frames?jobId=<id>`，`jobId` 是唯一任务定位参数。
- 只有完整完成且本地结果可读的任务可以创建工作区；进行中、失败、状态未知、已清理或损坏任务不得绕过守卫进入编辑。
- `/frames` 没有 `jobId` 时，展示本地可用的已完成任务列表，不静默选中任意任务。
- `jobId` 不存在、与帧资源不匹配或工作区版本无法读取时，展示明确原因和返回生成页/选择其他任务的恢复动作。
- 同一 `jobId` 默认只有一个工作区；重新进入恢复上次状态，不重复创建副本。

### 预览与逐帧检查

- 默认按工作区当前顺序、任务 `frameRate` 和 `loopMode` 播放；循环任务到尾帧后回到首帧，单次任务停在尾帧。
- 提供播放、暂停、重新开始、上一帧、下一帧和点击缩略图定位。
- 播放视图始终使用当前纳入序列且资源可读的帧；已移除帧不参与连续播放。
- 选中帧显示当前序号、原始 `sequenceIndex`、来源/替换版本、尺寸、格式、大小和审核状态。
- 高分辨率帧只在需要时解码；切换任务和卸载页面时释放对象 URL。

### 审核、筛选和非破坏性移除

- 每帧决策统一为 `pending`、`kept`、`removed`；筛选器支持全部、待审核、已保留和已移除。
- “移除”只将帧排除出当前工作序列并停止参与播放/交接，不删除原始任务帧 Blob。
- 已移除帧可恢复；恢复后回到原工作区相对位置附近，不能改变不可变的原始索引。
- 用户可逐帧标记保留，也可在不完成全部审核时继续预览；但生成可导出快照前必须满足可用性规则。
- MVP 允许工作区暂时少于两帧，方便用户整理；少于两帧时明确标记为“不可交接”，不阻止恢复操作。

### 稳定排序

- 展示顺序使用独立的有序帧 ID 列表表达，不改写 `providerIndex` 或 `sequenceIndex`。
- 支持拖拽排序，并提供键盘/按钮“前移、后移”作为可访问和移动端降级方案。
- 排序只影响当前纳入序列的帧；已移除帧保留稳定引用和恢复锚点。
- 任意编辑后预览顺序和后续交接顺序立即一致；刷新恢复后顺序不得漂移。

### 指定帧重试

- 用户从某一帧发起重试，系统冻结目标帧 ID、原始 `sequenceIndex`、父任务请求快照、相邻帧引用和当前工作区修订号。
- 重试状态至少区分 `submitting`、`running`、`candidate_ready`、`failed`、`status_unknown`、`accepted` 和 `discarded`。
- 重试候选不会自动覆盖当前帧；用户必须对比后明确选择“接受替换”或“放弃候选”。
- 接受替换只改变工作区中该槽位的当前资源引用，原始帧与重试历史继续保留；恢复原版不需要重新生成。
- 同一槽位同一时间只允许一个活动重试；重复点击必须幂等，`status_unknown` 不自动重提。
- 优先使用服务商声明的原生指定帧能力；当前 Seedance 无原生单帧能力时，MVP 建议采用“完整子任务重生成、只提取同一原始索引作为候选”的透明降级，并明确提示耗时与生成范围。
- 如果批准时不接受完整子任务降级，则本轮仍实现重试契约、能力检测和禁用说明，但真实指定帧重试不计入 MVP 完成；该范围差异必须在批准记录中写明。

### 工作区交接

- 工作区可交接的最低条件：至少 2 个纳入序列的帧；所有当前资源可读且尺寸/格式符合画布；没有活动或状态未知的重试；有序 ID 无重复、无悬空引用。
- `pending` 帧可以预览，但是否允许生成最终快照需在 P0 决策中冻结；建议要求所有纳入帧都已标记 `kept`，避免未审核素材被误导出。
- 交接快照包含工作区 ID/版本/修订号、来源任务 ID、最终帧顺序、每槽位来源版本、帧率、循环方式、画布、锚点和生成时间。
- 快照生成后保持不可变；继续编辑会增加工作区修订号并生成新的快照，不重写旧快照历史。

## 接口与数据流

### 领域对象建议

`FrameWorkspace` 至少表达：

- 稳定 `workspaceId`、schema 版本、`sourceJobId`、创建/更新时间和单调递增 `revision`。
- 来源交接快照：预设、帧率、循环方式、画布和锚点。
- 有序槽位 ID 列表，以及每个槽位对应的原始帧 ID、当前资源版本和原始 `sequenceIndex`。
- 每帧决策：`pending`、`kept` 或 `removed`。
- 选中帧、当前播放位置等仅影响恢复体验的 UI 状态；播放中/弹窗等瞬时状态不持久化。
- 重试尝试引用、当前候选引用和最后一次持久化修订号。

`FrameRevision` 至少表达：

- 稳定修订 ID、所属工作区和槽位、来源类型 `original` 或 `retry_candidate`。
- 原始帧 ID 或重试尝试 ID、资源引用、尺寸、格式、大小、创建时间和可读状态。
- 是否为该槽位当前采用版本；切换版本不删除其他修订。

`FrameRetryAttempt` 至少表达：

- 稳定尝试 ID、目标工作区/槽位/原始索引、父生成任务 ID 和客户端幂等 ID。
- 冻结的输入快照、执行模式 `native_single_frame` 或 `full_sequence_fallback`。
- 状态、时间、统一错误、候选修订引用和可恢复边界。
- 降级为完整子任务时的子 `GenerationJob` ID；不得把厂商私有请求写入工作区领域。

`FrameWorkspaceSnapshot` 至少表达：

- `workspaceId`、schema 版本、`revision`、`sourceJobId` 和快照创建时间。
- 按最终显示顺序排列的可读帧资源引用与连续 `outputIndex`。
- 每帧的来源帧 ID、采用修订 ID、原始 `sequenceIndex`、尺寸、格式和大小。
- 帧率、循环方式、画布和锚点；后续导出不能回读原任务猜测这些参数。

### 不变量

- 原始 `GenerationJob`、原始 `Frame`、`providerIndex` 和 `sequenceIndex` 不可变。
- 工作区顺序、审核决策和当前采用版本只能由 `FRAME_WORKSPACE` 拥有。
- 同一工作区不存在重复槽位 ID；每个活动槽位必须解析到一个可读资源。
- 接受重试候选是原子操作；保存失败时仍使用旧版本，不出现一半替换状态。
- 工作区引用的来源任务、原始帧、候选帧和活动重试不得被自动清理。
- 页面不持久化 Blob URL、data URL、完整签名 URL、凭据或服务商私有错误体。
- 所有用户编辑命令带预期 `revision`；旧页面或竞态保存不能覆盖较新的工作区。

### 存储边界

- 将 IndexedDB 从 v2 统一迁移到新版本，增加工作区元数据与候选资源所需 store；不能由功能模块自行打开不同数据库版本。
- 原始帧继续保存在 `frame-resources`；工作区只保存引用和编辑元数据。
- 原生单帧候选或需要脱离完整子任务保存的候选 Blob 使用独立工作区资源记录，避免污染原任务完整性。
- 清理器通过工作区引用图保护来源任务和候选资源；删除工作区后才允许按既有保留策略回收不再被引用的资源。
- 工作区元数据与候选采用关系必须事务性保存；容量不足时不改变当前可用序列。

### 模块数据流

```text
GENERATION.createFrameWorkspaceHandoff(jobId)
  → FRAME_WORKSPACE.loadOrCreate(handoff)
  → STORAGE.restoreWorkspace / restoreFrameBlobs
  → 用户预览与编辑命令
  → CORE 校验 revision 与不变量
  → STORAGE 原子保存 workspace revision
  → FRAME_WORKSPACE.createSnapshot()
  → EXPORT（后续模块）

指定帧重试：
FRAME_WORKSPACE.requestRetry(slotId, expectedRevision)
  → GENERATION 创建能力驱动的子任务
  → AI_GATEWAY / provider adapter
  → 候选资源安全落盘
  → FRAME_WORKSPACE 明确接受或放弃
```

## 失败场景

- URL 没有 `jobId`、`jobId` 不存在或指向非完成任务。
- 任务元数据仍在，但原始帧已被清理、Blob 损坏、MIME 不符或尺寸不一致。
- 工作区恢复时来源任务的 `frameIds` 与当前资源集合不一致。
- 排序后刷新出现顺序漂移，或误写原始 `sequenceIndex`。
- 移除当前播放帧后播放指针越界、停住或播放已移除帧。
- 所有帧被移除后页面无法恢复帧或产生空快照。
- 拖拽与自动保存竞态，旧顺序覆盖新顺序。
- 两个标签页编辑同一工作区，后保存者静默覆盖先保存者。
- 对象 URL 未释放导致多次进入任务后内存持续增长。
- 页面后台计时器累计漂移，回到前台后疯狂补帧或错序。
- 当前服务商没有原生单帧能力，页面却把完整重生成伪装成单帧请求。
- 重试提交超时但远端仍运行，用户重复点击产生多个候选任务。
- 完整子任务降级后错误使用显示顺序而不是原始 `sequenceIndex` 提取候选。
- 候选生成成功但 Blob 保存失败，页面错误地替换当前帧。
- 接受候选与清理同时发生，候选引用变为悬空。
- 工作区仍引用来源任务时，既有 20 个任务/30 天策略误清理其帧资源。
- v2→新版本迁移失败导致源图、历史生成任务或原始帧不可读。
- 快照含重复 `outputIndex`、已移除帧、未采用候选或不可读资源。

## 安全和性能影响

- **安全**：工作区只处理本地已持久化帧和统一重试用例；不读取或持久化 API Key，不直接调用服务商适配器。
- **隐私**：重试前明确说明会再次向所选服务商发送源图/任务上下文；本地纯预览、排序和移除不产生外部请求。
- **日志**：不得记录 Blob 内容、data URL、完整签名 URL、鉴权头、用户完整素材或服务商敏感错误体。
- **资源安全**：候选资源沿用域名、重定向、MIME、尺寸、体积、解码和超时校验；校验前不得进入工作区当前版本。
- **性能**：8/12 帧是 MVP 基线，但预览必须按需解码、限制对象 URL 缓存、避免每次状态变更重新读取全部 Blob。
- **播放准确性**：以单调时间基准计算目标帧，避免简单定时器累积误差；后台页面不持续高频绘制。
- **存储**：原始帧不复制；只有候选产生新 Blob。保存前预检配额，引用图防止误删，孤儿候选可回收。
- **兼容性**：拖拽不是唯一排序入口；Blob、IndexedDB、对象 URL、图片解码和页面可见性 API 建立目标浏览器基线。

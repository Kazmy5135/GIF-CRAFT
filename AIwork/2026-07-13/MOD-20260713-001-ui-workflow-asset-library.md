# UI 层级与资产工作流重构

## 目标

- 将当前按技术能力平铺的“生图 / 生成序列帧 / 序列帧工作区”导航，重组为用户任务导向的“新生成 / 库存”两个主要入口。
- 建立独立图库和序列帧库，让已生成资产可以被查找、复用并直接进入后续流程。
- 让一张已确认静态图可以创建多个相互独立的序列帧 ID，并让每个序列帧 ID 可追溯到来源静态图。
- 从序列帧调整工作区提供抽帧、帧率修改、序列帧预览和导出路径。
- 在工作区提供“重做”入口：回到对应静态图的确认步骤，按新生成流程创建新的序列帧 ID，不覆盖旧序列与旧工作区。
- 尽量复用已经跑通的源图、序列任务、帧工作区、预览和快照能力，避免因 UI 重排破坏既有任务状态机和不可变数据约束。

## 非目标

- 本 Draft 不实施代码、不修改数据库版本、不改变服务商调用。
- 不以视觉品牌、美术风格、动效或完整设计系统为本模块的主要范围；这些应在工作流和信息架构冻结后另行设计。
- 不建设账户、云同步、团队协作、计费或商业化能力。
- 不把“重做”实现成覆盖旧序列，也不把工作区指定帧候选重试与整条序列重做混为同一动作。
- 不在未确认“抽帧”含义和首期导出格式前承诺原视频重采样或具体编码器实现。

## 判级理由

本变更新增库存一级入口和资产库用户流程，重排应用导航及页面职责，并影响源图到序列 ID、序列库到工作区、工作区到导出的跨模块调用方向。帧率修改会改变工作区公开模型和快照元数据；若“抽帧”指从原视频重新采样，还会改变生成结果、媒体保存和服务端处理边界。因此属于大模块。

## 影响模块

- `APP`：一级导航、默认入口、路由层级、旧路由兼容与页面装配。
- `ASSET_LIBRARY`：新增规划模块，提供图库与序列帧库的查询、筛选、复用和跳转，不拥有或复制原始业务数据。
- `SOURCE_IMAGE`：将当前“生成表单 + 结果历史 + 确认为源图”拆成可被新生成流程和图库复用的公开用例。
- `GENERATION`：从只展示最新任务升级为可从指定静态图创建多个独立序列 ID，并向序列帧库暴露稳定摘要和来源关系。
- `FRAME_WORKSPACE`：新增整序列重做入口、可修改播放帧率的非破坏性工作区设置，并重新组织预览/抽帧/导出交接。
- `EXPORT`：从 Planned 进入本模块内的设计范围，消费不可变工作区快照并输出用户选择的首期格式。
- `STORAGE`：为资产库查询、重做关系、工作区帧率覆盖和导出记录提供索引或迁移；不得复制已有图片/帧 Blob。
- `CORE`：冻结序列资产 ID、来源关系、重做关系、工作区播放参数和导出输入契约。
- `AI_GATEWAY`：仅当“抽帧”被定义为从原始视频重新提取时受影响；如果只是工作区选帧，则不改变该模块。

## 当前行为

### 当前 UI 层级

- 左侧有四个同级入口：`/image` 生图、`/sequence` 生成序列帧、`/frames` 序列帧工作区、`/settings` 设置。
- 默认路由进入 `/image`。
- 页面按实现模块平铺，没有“新生成”和“库存”的上层任务分组。

### 当前生成与资产行为

- `/image` 同时承载文生图、图生图、本地上传、结果历史、源图确认、下载、复用参数和删除记录。
- 所有生成结果和本地上传都会保存为 `SourceImageAsset`；这已经具备图库的数据基础，但没有独立图库路由和库存入口。
- 源图确认通过全局 `currentSourceId` 表达，同时只能有一个“当前源图”；`/sequence` 只读取这个当前源图。
- 服务商支持时，一次生图可以返回多个候选，候选都会进入当前结果历史。
- `GenerationJob` 保存不可变源图快照，数据层可以保留多条任务；但 `SequenceContext` 和 `/sequence` 页面只恢复、展示最新一条任务。
- 完成任务的 `GenerationJob.id` 已经是稳定 ID，任务中已经保存来源静态图 ID，但当前 UI 没有把它正式定义并呈现为“序列帧 ID”。
- `/frames` 在没有 `jobId` 时会列出本地资源完整的已完成任务，具备序列帧库的部分查询基础，但它只是工作区任务选择器，不是完整库存页。

### 当前工作区与交付行为

- 工作区已经支持连续预览、逐帧审核、非破坏性移除/恢复、稳定排序、指定帧候选重试和不可变快照。
- 当前服务端在序列任务完成前，已经从 Seedance 返回的视频中均匀抽取 8/12 帧；原视频不是工作区的可重采样输入。
- 工作区帧率来自生成任务，领域模型中为只读来源元数据；用户不能在工作区修改。
- 连续预览已经嵌入工作区，不是独立的后置页面。
- `EXPORT` 尚未实现；当前流程只到“生成工作区快照”，没有实际文件导出。
- 工作区没有整条序列的“重做”入口。现有“指定帧重试”只为一个槽位生成候选；生成页的失败重试也不是从静态图重新配置并创建全新序列的流程。

## 目标行为

### 目标主流程

```text
入口：新生成
  → 确认静态图（可生成新图，也可从图库选择）
  → 序列帧生成（同一静态图可创建多个独立序列帧 ID）
  → 序列帧调整工作区
  → 抽帧 / 帧率修改 / 序列帧预览
  → 导出

入口：库存
  ├→ 图库 → 选择并确认静态图 → 新生成流程
  └→ 序列帧库 → 选择序列帧 ID → 对应工作区

工作区：重做
  → 定位当前序列对应的静态图
  → 回到确认静态图
  → 重新配置并创建新的序列帧 ID
  → 旧序列、旧工作区和旧快照保持不变
```

### 建议 UI 层级

| 层级 | 建议入口 | 页面职责 |
|---|---|---|
| 一级主入口 | 新生成 | 承载“确认静态图 → 配置序列 → 生成/跟踪 → 进入工作区”的引导流程 |
| 一级主入口 | 库存 | 承载图库和序列帧库两个二级视图 |
| 二级库存 | 图库 | 展示所有可用 `SourceImageAsset`，支持确认、复用、删除和发起新生成 |
| 二级库存 | 序列帧库 | 展示序列 ID、来源图、预设、状态、帧数、FPS、更新时间和工作区入口 |
| 任务工作区 | 序列帧调整 | 处理审核、排序、候选、抽帧语义、帧率覆盖、预览、快照、重做和导出 |
| 工具入口 | 设置 | 保留为辅助入口，不属于生产主流程步骤 |

路由建议仅作为 Draft 基线：新增 `/create`、`/library/images`、`/library/sequences`、`/workspace/:sequenceId` 和 `/export/:snapshotId`；旧 `/image`、`/sequence`、`/frames?jobId=` 在过渡期重定向或兼容，避免刷新和旧书签失效。

### 资产与 ID 建议

- 图库条目继续使用 `SourceImageAsset.id`，不复制图片字节。
- Draft 推荐直接把一个完整 `GenerationJob.id` 定义为一个“序列帧 ID”，避免新增一套与任务重复的 ID。只有当序列需要跨多次任务聚合、重命名或版本化时，才新增独立 `SequenceAsset` 聚合。
- 一个 `SourceImageAsset.id` 可以关联多个 `GenerationJob.id`；每次新生成或工作区重做都创建新 ID。
- 整序列重做建议增加独立的 `redoOfJobId` 或等价关系字段，不复用“失败重试”的 `parentJobId` 语义。
- 资产库优先作为现有源图仓储、任务仓储和工作区仓储之上的查询/展示模块，不成为第二份数据所有者。
- 失败、取消、状态未知的任务默认属于“任务历史”，是否也显示在“序列帧库”需在批准前决定；Draft 推荐序列帧库主视图只把结果完整且本地资源可读的完成任务视为可用资产，同时提供任务状态筛选。

### 工作区后处理建议

- 帧率修改采用非破坏性覆盖：保留生成任务原始 FPS，在工作区保存 `playbackFrameRate`，预览与新快照使用覆盖值，旧快照不变。
- 序列帧预览复用现有连续预览控制器，可作为工作区模式或面板，不必先拆成独立持久化模块。
- 导出只消费不可变工作区快照；修改帧、排序或 FPS 后必须生成新快照，已有导出记录仍指向原快照。
- “抽帧”必须先冻结产品语义：
  - 如果表示从现有 8/12 帧中选取、移除或重新排序，当前工作区已具备大部分能力，主要是改名、批量操作和交互重组。
  - 如果表示从 Seedance 原视频按新采样规则重新抽取，则需要保留/持久化原视频、增加媒体采样参数、服务端重处理 API、存储清理和安全校验，属于明显扩大的后端范围。

## 当前流程与目标流程差异

| # | 目标节点/关系 | 当前状态 | 差异类型 | 需要修改 |
|---|---|---|---|---|
| 1 | “新生成 / 库存”两个一级入口 | 生图、生成序列帧、工作区、设置四个同级页签 | 重构 | 重做应用信息架构、默认入口、侧栏和路由兼容 |
| 2 | 独立“确认静态图”步骤 | 确认动作嵌在生图结果卡片；序列页只读取全局当前源图 | 重构 | 提取可复用确认用例和步骤状态，支持新生成与图库共同进入 |
| 3 | 生图后自动进入图库 | 结果已持久化在生图页历史中 | 已有数据、缺 UI | 将结果历史提升为图库；保留多候选和本地上传 |
| 4 | 库存 → 图库 | 没有库存路由；`/image` 兼作历史 | 新增页面 | 新增图库列表、筛选、预览、确认、复用和删除入口 |
| 5 | 一张静态图可生成多个序列 ID | 仓储可保留多任务，页面只呈现最新任务 | 部分已有 | 暴露任务列表和来源关系；明确串行多次还是一次批量创建多个 |
| 6 | 生成时创建“序列帧 ID” | 已有稳定 `GenerationJob.id`，UI 只称“任务 ID” | 语义/UI 调整 | 确认是否复用 job ID；统一文案、路由参数和库条目键 |
| 7 | 库存 → 序列帧库 | 仅 `/frames` 任务选择器列出可用完成任务 | 新增页面 | 建立序列帧资产列表、状态/来源摘要、工作区和重做入口 |
| 8 | 序列帧库 → 指定工作区 | 已支持 `/frames?jobId=...` | 基本已有 | 从库直接导航；建议迁移到稳定路径并保留旧链接兼容 |
| 9 | 工作区 → 序列帧预览 | 工作区已经内嵌连续预览 | 已实现、需重组 | 按新层级重排为明确功能区，无需重写播放内核 |
| 10 | 工作区 → 帧率修改 | 生成前可选 FPS，工作区 FPS 只读 | 新增领域能力 | 增加非破坏性 FPS 覆盖、revision、持久化、快照和测试 |
| 11 | 工作区 → 抽帧 | 服务端在任务完成前固定抽 8/12 帧；工作区支持选帧/移除 | 定义冲突 | 先确定“选帧”还是“从原视频重采样”，再冻结范围 |
| 12 | 工作区后导出 | 只有不可变快照，`EXPORT` 未实现 | 新增模块 | 确定首期格式，建立导出用例、状态、编码、下载和失败恢复 |
| 13 | 工作区整序列重做 | 只有指定帧候选重试、失败任务重试和生成页返回替换源图 | 新增流程 | 用来源图 ID 回到确认步骤，预填上下文，新建序列 ID 并记录 lineage |
| 14 | 旧资产不被重做覆盖 | 原任务/原帧/工作区本已不可变 | 可复用不变量 | 将新重做流程接入现有不可变模型，禁止改写旧记录 |
| 15 | 库存可持续恢复 | 当前 IndexedDB 有清理保留策略，完成任务帧可能被清理 | 风险 | 定义库可见性、已清理状态、恢复动作和引用保护策略 |

## 接口与数据流

```text
APP
  ├─ NewGenerationFlow
  │    ├─ SOURCE_IMAGE.confirm/select/generate
  │    └─ GENERATION.create/track
  ├─ ASSET_LIBRARY
  │    ├─ query SOURCE_IMAGE assets
  │    └─ query GENERATION + FRAME_WORKSPACE summaries
  └─ FRAME_WORKSPACE
       ├─ load by sequenceId
       ├─ edit non-destructively
       ├─ update playbackFrameRate
       ├─ create immutable snapshot
       ├─ start redo from sourceImageId
       └─ hand snapshot to EXPORT
```

- `ASSET_LIBRARY` 只能调用各模块的公开查询接口，不直接拼接 IndexedDB store，不复制 Blob，也不修改任务状态机。
- `NewGenerationFlow` 应显式持有本次流程选择的 `sourceImageId`，避免仅依赖一个不可见的全局 `currentSourceId`。
- `GenerationJob.request.source.id` 继续作为序列到静态图的稳定追溯键。
- `FRAME_WORKSPACE` 保持一条序列任务对应一个工作区；整序列重做创建另一条任务和另一工作区。
- 帧率覆盖不得回写 `GenerationJob.request.effectiveParameters.frameRate`，只改变新工作区 revision 和后续快照。
- `EXPORT` 输入必须是已持久化且资源可读的 `FrameWorkspaceSnapshot`，不能直接读取页面临时排序或对象 URL。

## 失败场景

- 旧 URL 或刷新被重定向到错误步骤，导致已选源图、任务或工作区上下文丢失。
- 全局 `currentSourceId` 与当前新生成流程选择不一致，生成了错误源图对应的序列。
- 同一静态图创建多个序列时错误复用幂等 ID，导致任务被合并或互相覆盖。
- 序列帧库把失败任务、资源已清理任务或部分结果误标为可用资产。
- 工作区重做误用原任务 ID，覆盖旧任务、旧工作区或快照。
- 对应静态图已删除、损坏或不可读，重做仍允许提交。
- 帧率覆盖回写原始生成任务，使历史预览或旧导出无法复现。
- 多标签页同时修改 FPS 或帧顺序，旧 revision 覆盖新 revision。
- “抽帧”被误解为已有选帧，实际用户需要从原视频重新采样，造成范围和数据能力不匹配。
- 如果需要原视频重采样，原视频未持久化、已过期、过大或未通过媒体安全校验。
- 导出直接读取临时对象 URL 或页面内存状态，刷新后文件不可复现。
- 导出编码占用过高导致浏览器冻结、内存不足或生成半成品文件。
- 清理策略为了容量删除仍在图库、序列库、工作区、快照或导出记录中被引用的资源。

## 安全和性能影响

- 安全：库存页不得展示明文凭据、完整签名 URL、data URL 或原始 Blob；所有外发动作仍需在明确提交后发生。
- 安全：如果启用原视频重采样，必须继续执行下载域名、重定向、MIME、编码、尺寸、体积和超时校验。
- 隐私：图库和序列帧库均为本地 IndexedDB 资产视图，界面需明确数据保存位置和清理影响。
- 性能：列表只加载摘要和受控缩略图，不应在首屏读取/解码所有原图与全部帧 Blob。
- 性能：导出应按格式评估 Web Worker、流式打包、内存上限和取消策略。
- 兼容性：IndexedDB v4 数据必须可迁移；旧任务、工作区和快照不得因新增可选字段变得不可读。

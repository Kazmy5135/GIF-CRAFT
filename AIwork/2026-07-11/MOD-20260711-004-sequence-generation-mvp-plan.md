# 序列帧生成模块 MVP 开发清单

## 目标

- 在现有“已确认源图”基础上，实现角色/场景序列帧的配置、请求预览、提交、查询、取消、恢复和结果接收。
- 固化 `SequencePreset`、`GenerationJob`、`Frame`、有效参数和任务状态契约，确保任务可追踪、可恢复、可复现。
- 接通一个经过真实能力验证的图生序列帧服务商，并将其协议、字段和输出形态收敛在 `AI_GATEWAY`。
- 输出具有稳定索引、统一画布、帧率、循环方式和对齐信息的有序帧集合，交给 `FRAME_WORKSPACE`。
- 将 MVP 拆成可执行、可验收、带依赖关系的开发 List，作为批准后的实施基线。

## 非目标

- 不在本模块实现帧删除、拖拽排序、逐帧审核和指定帧重试；这些属于 `FRAME_WORKSPACE`。
- 不在本模块实现图片包、GIF、WebP 或视频导出；这些属于 `EXPORT`。
- 不实现高级时间线、关键帧编辑、骨骼动画、蒙版、局部重绘或专业图片编辑。
- MVP 不支持多张源图、多任务批处理、自动跨服务商降级或并行生成多个方案。
- 不开放公共工程底词直接编辑；历史底词只能通过新版本演进。
- 不在首期承诺用算法自动修复身份漂移、肢体异常、镜头漂移或循环接缝，只检测、呈现并提供明确的后续重试入口。
- 不在本模块补齐完整 `PROJECT` 项目管理；只保存生成任务自身可恢复、可复现所需的最小快照。
- 本轮不实现任何商业化或计费能力，包括价格展示、费用确认、额度余额、预算提醒、消费台账、部门分摊、支付和订阅。

## 判级理由

本需求新增完整的外部生成流程、任务状态机、序列领域模型、持久化结构、AI Gateway 序列能力和服务商适配，涉及跨模块接口、并发、幂等、资源容量、隐私和错误恢复，属于大模块。

## 影响模块

- `GENERATION`：主模块，负责参数配置、任务编排、状态、进度、取消、安全重试和结果接收。
- `AI_GATEWAY`：新增图生序列帧能力声明、统一序列接口、服务商任务查询、取消和结果归一化。
- `CORE`：新增预设、序列参数、生成任务、帧、状态转换和统一错误契约。
- `STORAGE`：新增序列任务和帧资源存储，并处理当前 IndexedDB v1 的统一升级问题。
- `APP`：将 `/sequence` 占位页替换为真实页面并装配生成模块 Provider/用例。
- `SOURCE_IMAGE`：只提供已确认且可读取的不可变输入快照，不承担序列生成状态。
- `FRAME_WORKSPACE`：只接收完成任务的稳定帧集合；本次定义交接契约，不实施其编辑能力。

## 当前行为

- `/sequence` 仍是占位页，没有预设、参数表单、生成任务或结果展示。
- `SourceImageAsset` 已能从 Gorilla Banana、Image2 或本地上传得到，并通过 `currentSourceId` 标记当前源图；但当前对象没有独立的 `confirmedAt` 和资源可读性复核结果。
- `ProviderCapabilities` 只声明文生图、图生图、透明背景、候选数量和取消能力，尚未声明图生序列帧能力及序列参数范围。
- 代理只有 `/api/source-images/generate`，没有序列任务提交、查询、取消和结果接口。
- `CORE` 尚无 `SequencePreset`、`SequenceGenerationRequest`、`GenerationJob` 和 `Frame` 实现。
- IndexedDB `gif-craft` 当前为 v1，只有 `source-images` 存储。若新仓储单独用更高版本打开而旧仓储仍固定 v1，会产生版本不一致风险，实施前必须统一数据库打开与迁移入口。
- 当前请求幂等只保存在代理进程内存中，适用于短时源图请求，不足以直接承担长耗时异步序列任务的跨刷新恢复。
- 首个图生序列帧服务商、真实输出形态、参数范围、查询协议和取消能力尚未选定。

## 目标行为

### MVP 用户流程

1. 用户进入“生成序列帧”，系统读取当前已确认源图。
2. 没有源图、源图未确认或资源不可读取时，页面阻止继续并提供返回“生图”页的入口。
3. 用户选择“角色”或“场景”。角色继续选择“待机 / 攻击 / 其他”，场景使用通用固定镜头预设。
4. 页面载入对应预设默认值，用户填写动作或场景运动描述，并调整允许修改的帧数、帧率、循环方式、画布/宽高比和随机种子。
5. 系统编译版本化提示词，校验用户描述不能覆盖固定画布、单一主体、稳定锚点等硬约束。
6. 系统根据所选服务商能力计算“最终有效参数”，明确展示映射、预计输出和源图外发提示。
7. 用户点击“开始生成”后提交一次带客户端幂等 ID 的任务；按钮锁定，重复点击不得创建第二个外部生成任务。
8. 页面展示排队、提交、生成、处理、成功、失败、状态未知或取消状态，并按能力轮询或取消。
9. 刷新页面后恢复任务快照，并继续查询尚未结束的任务；不能查询时明确显示恢复限制。
10. 成功后校验帧数量、顺序、尺寸、格式和可读取性，保存稳定帧索引并交给帧工作区。

### MVP 范围分界

| 范围 | 本模块 MVP | 后续模块/阶段 |
|---|---|---|
| 源图检查 | 已确认、可读取、冻结快照 | 源图编辑和重新生成由 `SOURCE_IMAGE` 负责 |
| 预设 | 角色待机、攻击、其他；场景通用 | 更多动作、用户自定义预设 |
| 参数 | 帧数、帧率、循环、画布/宽高比、锚点、随机种子 | 高级连续性、关键帧、镜头曲线 |
| 任务 | 提交、查询、进度、取消能力声明、恢复、安全整任务重试 | 队列调度、多任务并发、跨服务商降级 |
| 结果 | 稳定索引、统一元数据、最小结果缩略/摘要 | 删除、排序、逐帧审核、局部重试由 `FRAME_WORKSPACE` 负责 |
| 导出 | 只提供交接数据 | 图片包及动画格式由 `EXPORT` 负责 |
| 商业化/计费 | 不实现、不采集金额或余额数据 | 不在当前路线图中 |

### 页面信息结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 生成序列帧                                                  │
├───────────────────────┬──────────────────────────────────────┤
│ 已确认源图            │ 请求预览 / 当前任务                  │
│ [预览] [返回替换]     │ 编译提示词摘要                       │
│                       │ 请求参数 → 最终有效参数              │
│ 类型：角色 / 场景     │ 提交和隐私提示                       │
│ 动作：待机/攻击/其他  │ [开始生成]                           │
│ 运动描述              │                                      │
│ 帧数 / 帧率 / 循环    ├──────────────────────────────────────┤
│ 画布 / 随机种子       │ 状态、进度、耗时、错误和恢复动作     │
│ 服务商                │ 成功后帧摘要与“进入帧工作区”         │
└───────────────────────┴──────────────────────────────────────┘
```

- 配置区和任务区分离；已经提交的任务不随表单继续编辑而改变。
- 参数发生服务商映射时，以“请求值 → 有效值”逐项展示，不只显示最终值。
- 服务商不支持取消时不显示伪取消；不支持真实进度时显示阶段和已耗时，不伪造百分比。
- `status_unknown` 不显示普通“重新生成”按钮，必须先查询或对账，避免创建重复任务和混淆结果。

## 接口与数据流

### 领域对象

`SequencePreset` 至少表达：

- 稳定 `id`、版本、类型、动作子类和显示名称。
- 默认帧数、帧率、循环方式、画布/宽高比、锚点和对齐规则。
- 公共底词、类型底词、动作底词和负向约束的 ID/版本引用。
- 可由用户修改的字段、允许范围和硬约束。

`SequenceGenerationRequest` 至少表达：

- `clientRequestId` 和内部草稿 ID。
- 已确认源图的稳定 ID、确认时间、资源引用、MIME、宽高和内容快照标识。
- 预设 ID/版本、提示词层版本、用户描述和编译提示词快照。
- 用户请求参数、服务商映射后的最终有效参数和映射说明。
- 服务商配置引用；扩展参数与核心字段隔离。

`GenerationJob` 至少表达：

- 内部任务 ID、客户端幂等 ID、外部任务引用及服务商。
- 不可变输入快照和有效参数。
- 当前状态、真实进度或阶段、创建/提交/更新/完成时间。
- 查询游标或恢复信息、最近错误、重试次数和父任务 ID。
- 结果帧 ID 列表及结果完整性状态。

`Frame` 至少表达：

- 稳定帧 ID、任务 ID、原始索引和显示索引。
- 资源引用、MIME、宽高、大小和可读取状态。
- 服务商返回的时间点或序号；不存在时按归一化步骤生成。
- 生成来源、创建时间和最小元数据；不得保存密钥或完整签名 URL。

### 状态机候选

```text
draft → validating → ready → submitting → queued → generating → processing → completed
                      │          │          │           │
                      │          ├──────────┴───────────┴→ failed
                      │          └────────────────────────→ status_unknown
                      └───────────────────────────────────→ cancelled

failed → retrying → submitting
queued / generating / processing → cancelling → cancelled
```

- `status_unknown` 不是 `failed`；未完成对账前不能自动创建新的外部生成任务。
- `completed` 只有在结果数量、索引、资源可读性和必要元数据校验完成后进入。
- 服务商返回部分帧时进入明确的“不完整结果”错误或结果状态，不能伪装成完整成功。
- 重试创建新任务并保存 `parentJobId`，不回写旧任务的不可变输入和历史状态。

### 提示词合成

```text
game.sequence.common.v1
  + game.sequence.character.v1 / game.sequence.scene.v1
  + idle.v1 / attack.v1 / other.v1（角色适用）
  + 用户动作或场景运动描述
  + game.sequence.negative.v1
```

- 结构化参数是帧数、帧率、循环、锚点和画布的唯一事实源。
- 用户描述可以补充动作、情绪、节奏和局部环境运动，不能覆盖身份一致、固定镜头、稳定画布和对齐规则。
- 任务同时保存提示词层 ID/版本和最终编译文本；升级模板不会改变历史任务。

### AI Gateway 统一能力

```text
getSequenceCapabilities(provider) → SequenceProviderCapabilities
createSequenceJob(request) → SequenceJobReceipt
getSequenceJob(jobRef) → SequenceJobSnapshot
cancelSequenceJob(jobRef) → CancelResult
getSequenceResult(jobRef) → SequenceGenerationResult
```

`SequenceProviderCapabilities` 至少声明：

- 是否支持图生序列、异步查询、取消、随机种子和真实进度。
- 支持的输入格式、输入尺寸、帧数、帧率、时长、宽高比和输出格式。
- 输出形态：独立帧、精灵表、视频或其他；以及能否无损归一化为有序帧。
- 并发、速率限制、结果保留期和任务查询限制。

建议的同源代理接口：

| 方法 | 路径 | 作用 |
|---|---|---|
| `GET` | `/api/providers` | 扩展返回序列能力摘要，不下发密钥 |
| `POST` | `/api/sequence-jobs` | 校验并提交一次序列任务 |
| `GET` | `/api/sequence-jobs/:jobId` | 查询统一状态和进度 |
| `POST` | `/api/sequence-jobs/:jobId/cancel` | 能力支持时取消任务 |
| `GET` | `/api/sequence-jobs/:jobId/result` | 返回已归一化的有序帧结果 |

最终路径可在批准后按现有代理命名约定微调，但必须保持资源语义和统一错误结构。

### 输出归一化门槛

- 优先选择原生返回独立有序帧且能查询异步状态的服务商。
- 返回精灵表时，必须有可靠行列、帧数和切割顺序元数据；不允许仅靠图片尺寸猜测。
- 返回视频时，需要额外的解码/抽帧、时间点映射和浏览器/服务端性能方案。是否接受“视频转帧”作为 MVP 服务商路径，必须在服务商选型时单独批准。
- 所有输出最终都要通过统一校验：数量、稳定索引、相同尺寸、允许格式、可读取、无空帧和资源可持久化。

### 存储与恢复

- 抽取统一 IndexedDB 打开和迁移入口，避免 `source-images` 仓储与新序列仓储使用不同数据库版本。
- 至少新增任务存储和帧资源存储；任务元数据与大体积图片资源分离。
- 优先保存 `Blob`/受控缓存引用而不是重复的 base64 字符串，降低约 33% 的编码膨胀；具体迁移策略批准后确定。
- 提交前检查存储可用性，成功后再标记任务结果可恢复；容量不足必须保留任务元数据并给出清理或下载建议。
- 记录资源保留策略、任务清理顺序和当前/历史任务保护规则，禁止清理正在生成或仍被工作区引用的帧。
- 浏览器刷新恢复和代理进程重启恢复分别定义；不能把“刷新可恢复”宣传为“服务重启可恢复”。

## 失败场景

- 当前源图记录存在但图片字节已丢失、损坏或无法读取。
- 服务商只支持文生图/图生图，不支持图生序列，页面却允许提交。
- 请求参数被服务商静默映射，用户在提交后才发现帧数或时长变化。
- 提交超时后实际已受理，用户重试导致重复任务和结果混淆。
- 查询响应倒序或缓存导致任务状态从后期回退到前期。
- 服务商返回部分帧、重复索引、缺失索引、不同尺寸或不可下载资源。
- 临时资源 URL 在持久化前过期，任务标记成功但工作区无法读取。
- 页面刷新后丢失外部任务引用，无法继续查询。
- 代理重启导致内存幂等映射丢失，恢复能力与页面承诺不一致。
- IndexedDB 版本分散导致旧源图仓储出现 `VersionError`。
- 大量 base64 帧造成存储膨胀、主线程卡顿或容量耗尽。
- 用户描述覆盖固定镜头、稳定锚点或单一主体约束。
- 取消请求与完成响应竞态，已完成结果被错误标记取消。
- 真实进度不可用时伪造百分比，误导用户判断卡死。

## 安全和性能影响

- **安全**：序列生成继续使用服务端代理；H5 不接收服务端 API Key。所有请求、查询和资源下载执行域名、重定向、MIME、体积和超时限制。
- **隐私**：提交前明确显示源图将发送至哪个服务商；本地源图只有用户明确提交后才上传。
- **日志**：Authorization、Token、Cookie、完整签名 URL、data URL、原始图片字节和服务商敏感错误体必须脱敏。
- **可靠性**：生成和重试都是显式操作；未知状态不自动重提；幂等只保护相同请求，不把不同参数误合并。
- **内部工具边界**：不统计金额、余额、预算或部门用量，不提供价格、账单、支付和订阅界面。
- **性能**：序列帧数量远大于源图候选，需控制并发下载、解码和缩略图加载，避免一次把全部原图解码到主线程。
- **存储**：实施前实测 8/12 帧典型体积，制定配额预警、Blob 保存和清理策略。
- **兼容性**：图片解码、IndexedDB、Blob 和对象 URL 在目标浏览器建立最低兼容基线。

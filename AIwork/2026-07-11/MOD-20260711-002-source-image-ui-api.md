# 生图模块 UI 与双服务商适配

## 目标

- 实现“生图”页的文生图、图生图和本地图片直接使用三条入口。
- 针对 AIGC 生图常见问题提供输入校验、能力匹配、状态恢复和结果追溯。
- 通过统一 `AI_GATEWAY` 接入两个图像服务商，不在页面内写厂商协议。
- 提供调用所需的最小设置能力，包括服务商、模型、连接方式和提示词模板。
- 生成或上传的结果统一转换为 `SourceImageAsset`，供后续序列帧模块使用。

## 非目标

- 不在本模块实现图生序列帧、帧调整或导出。
- 不实现 Photoshop 类图层、蒙版、画笔、局部重绘或复杂图片编辑器。
- 不承诺通过提示词完全消除 AIGC 的角色一致性和构图随机性。
- 不把真实 API Key 写入仓库、URL、日志或普通项目草稿。
- 不在 API 身份、Base URL、鉴权和请求协议未确认前编写具体适配器。

## 判级理由

本变更实现首个完整用户功能，新增 AI 服务商适配、API 凭据处理、计费任务状态、图片上传与持久化、提示词模板和结果选择，涉及安全、成本和跨模块契约，属于大模块。

## 影响模块

- `SOURCE_IMAGE`：实现三种源图入口、生成任务、结果历史和源图确认；首期不实现精细错误分类面板。
- `AI_GATEWAY`：新增服务商能力发现、配置校验、文生图和图生图统一契约。
- `SETTINGS`：新增 API 连接配置和基础提示词模板管理。
- `PROJECT`：关联当前确认源图和生成历史。
- `STORAGE`：保存非敏感设置、提示词覆盖、生成元数据和本地图片引用。
- `CORE`：固定 `SourceImageJob`、`SourceImageAsset`、提示词快照和统一错误。
- `APP`：挂载 `/image` 和最小 `/settings` 页面。

## 当前行为

- 源图流程和统一领域概念已经批准，但没有代码或 UI。
- API 已确认是 Google Gemini 官方 API 与 OpenAI GPT Image 2 官方 API。
- 项目没有 API 配置、凭据策略、提示词模板存储和生成历史实现。

## 目标行为

### 生图页布局

```text
┌─────────────────────────────────────────────────────────────┐
│ 生图                                                        │
├──────────────────────┬──────────────────────────────────────┤
│ 输入方式             │ 当前结果                             │
│ ○ 文生图             │ 图片预览                             │
│ ○ 图生图             │ 生成状态 / 错误 / 重试              │
│ ○ 直接使用本地图片   │ 确认为序列帧源图                    │
│                      │                                      │
│ 服务商 / 模型        ├──────────────────────────────────────┤
│ 参考图上传           │ 本次会话与项目生成历史               │
│ 用户提示词           │                                      │
│ 画布 / 质量 / 数量   │                                      │
│ 最终提示词预览       │                                      │
│ [生成 / 使用图片]    │                                      │
└──────────────────────┴──────────────────────────────────────┘
```

- “直接使用本地图片”不调用生图 API，校验后可以直接确认。
- 文生图隐藏参考图字段；图生图必须先选择参考图。
- 页面根据服务商能力显示或禁用参数，不能提交厂商不支持的组合。
- 结果不会自动覆盖当前源图，必须由用户点击确认。

## 常见问题与对应功能

| 常见问题 | 必须开发的功能 |
|---|---|
| 提示词过短、主体和用途不明确 | 提示词必填、用途提示、基础模板合成、最终提示词预览 |
| 用户提示词与工程底词冲突 | 分层提示词、硬约束保护、冲突提示、保存编译快照 |
| 文生图和图生图参数混用 | 明确输入模式，按模式显示字段并在提交前校验 |
| 参考图格式错误、损坏或过大 | 拖放/选择上传、类型/容量/尺寸/解码校验、替换和删除 |
| EXIF 方向导致预览与上传不一致 | 上传后按实际解码方向预览并记录规范化尺寸 |
| 模型不支持尺寸、质量或透明背景 | 服务商能力矩阵、只展示支持选项、提交前显示有效参数 |
| 图生图对原图保持程度不可预测 | 提供“保留原图/适度调整/自由重绘”意图；仅在适配器能可靠映射时启用 |
| 角色身份、服装、构图发生漂移 | 游戏资产基础模板、参考图预览、明确一致性提示和重新生成入口 |
| 生成耗时且用户误以为卡死 | 明确任务状态、耗时提示、进度或轮询状态、可取消能力说明 |
| 用户连续点击造成重复计费 | 提交锁、客户端幂等 ID、计费请求确认和按钮防抖 |
| 网络超时后不知道是否已扣费 | 区分“提交失败”和“状态未知”，状态未知先查询，不自动重复提交 |
| 代理未配置、API Key 无效或服务商拒绝 | 设置页显示服务商“已配置/未配置”，生成失败时显示通用脱敏摘要 |
| API 请求失败 | 首期显示通用失败状态、服务商名称和脱敏错误摘要，不开发独立的鉴权/额度/限流/安全分类面板 |
| API 返回文字但没有图片 | 验证响应中存在可读取图片；保留服务商说明并阻止确认 |
| 临时图片 URL 过期 | 收到结果后尽快转为受控本地或代理缓存引用，并标记有效性 |
| 重新生成覆盖满意结果 | 每次尝试生成新历史项，明确选择当前结果，不静默覆盖 |
| 多张候选难以区分 | 结果画廊、单选确认、放大预览、下载和元数据查看 |
| 页面刷新丢失任务 | 保存任务 ID 和非敏感状态；支持服务商查询时恢复轮询 |
| 无法复现历史图片 | 保存服务商、模型、输入模式、提示词快照、参考图引用和有效参数 |
| 密钥被浏览器持久化或泄漏 | 默认只存内存会话；日志脱敏；代理模式不把服务端密钥下发 H5 |

## 功能清单

### 1. 输入模式

- 文生图：用户提示词、服务商、模型、画布、质量和候选数量。
- 图生图：在文生图字段基础上增加一张参考图和改动意图。
- 直接上传：只执行图片校验和确认，不显示服务商或计费操作。
- 切换模式时保留兼容字段，但不把隐藏字段意外提交给 API。

### 2. 提示词系统

- 内置不可变的默认模板版本。
- 设置页允许创建“用户覆盖版本”，支持启用、编辑、重置默认和查看差异。
- 内置版本不直接修改，避免升级后历史任务不可复现。
- 最终提示词按“内置基础模板＋用户覆盖模板＋用户描述＋负向约束”编译。
- 提交前显示最终提示词和模板版本。
- 每次任务保存实际编译文本，不能只保存模板 ID。

### 3. 图片输入校验

- 支持点击、拖放、替换和删除参考图。
- 校验 MIME、扩展名、文件大小、解码能力、宽高和像素总量。
- 使用浏览器实际解码结果生成预览，不能只相信文件扩展名。
- 不静默压缩或裁切；需要规范化时先展示原始值和目标值并由用户确认。
- 错误图片不能进入任务提交或源图确认。

首期具体格式和限制根据两个 API 的共同能力确定，在适配器身份确认前不硬编码。

### 4. 服务商和模型选择

- 设置页创建连接配置，生图页只选择已通过基本校验的连接。
- 每个适配器声明文生图、图生图、尺寸、质量、数量、透明背景、取消和状态查询能力。
- 生图页以统一名称展示能力，不显示无意义的厂商字段。
- 高级厂商参数可以折叠展示，但必须与核心字段隔离。
- 服务商切换后重新校验所有参数，并明确显示被调整的有效值。

### 5. 任务生命周期

```text
draft → validating → submitting → generating → succeeded
                  │          │          │
                  └──────────┴──────────┴→ failed
                                       └→ status_unknown
                                       └→ cancelled（服务商支持时）
```

- `submitting` 后禁止重复点击。
- 只有明确未被服务商接受的请求可以安全重新提交。
- `status_unknown` 优先查询原任务，不能自动创建新计费任务。
- 自动重试只用于确定安全的查询请求；生成提交默认由用户确认重试。
- 首期统一显示通用失败状态与脱敏摘要；精细错误分类和针对性修复面板推迟到后续模块。

### 6. 结果画廊与确认

- 支持一个或多个候选结果，也支持多次生成尝试累积历史。
- 每个结果显示图片、来源模式、服务商、模型、时间、规格和任务状态。
- 提供放大预览、下载、重新生成、删除本地记录和“确认为源图”。
- 确认操作创建或更新统一 `SourceImageAsset`，但不删除其他历史结果。
- 生成结果过期或无法读取时禁止确认，并提供恢复或重新生成说明。

### 7. 最小设置页

- API 连接：显示名称、服务商类型、官方 Base URL、模型 ID和服务端配置状态。
- 凭据：只从代理服务的环境变量或密钥管理读取，不在 H5 中输入、返回或持久化。
- 连接检查：优先使用非计费能力校验；如果只能通过真实生成验证，必须明确提示会产生费用。
- 基础提示词模板：查看内置版本、创建用户覆盖、重置和启用。
- 设置只保存非敏感配置和凭据引用，不能保存明文密钥到项目草稿。

## 统一接口与数据流

### 服务商适配器

```text
getCapabilities(config) → ProviderCapabilities
validateConfig(config) → ValidationResult
generateImageFromText(request) → SourceImageJob
generateImageFromImage(request) → SourceImageJob
getSourceImageJob(jobId) → SourceImageJob
cancelSourceImageJob(jobId) → CancelResult
```

- 同步返回图片的服务商仍映射为统一任务，并直接进入 `succeeded`。
- 不支持查询或取消时，能力矩阵必须明确为 `false`。
- 首期统一错误只区分 `request_failed`、`status_unknown` 和 `no_valid_image`，并携带脱敏服务商摘要；更细分类不进入本次范围。

### 配置对象

`ProviderProfile` 至少表达：

- 内部 ID 和用户显示名称。
- 服务商适配器类型。
- Base URL 和模型 ID。
- 凭据引用，不包含持久化明文密钥。
- 能力快照与最近验证时间。

`PromptTemplateOverride` 至少表达：

- 模板 ID、作用域、基于的内置版本。
- 用户覆盖内容和是否启用。
- 创建时间、更新时间和版本号。

### 生图请求

统一请求至少表达：

- `mode`：`text_to_image` 或 `image_to_image`。
- 用户提示词、基础模板快照和负向约束。
- 图生图参考图片引用及用户改动意图。
- 请求画布、质量、候选数量和输出格式。
- 客户端幂等 ID。
- 与统一字段隔离的服务商扩展参数。

### 生图结果

统一结果至少表达：

- 内部与外部任务 ID、状态和时间。
- 图片列表及每张图片的 MIME、宽高、大小和可读取引用。
- 服务商、模型、输入模式和有效参数。
- 编译提示词与参考图快照引用。
- 统一错误、服务商错误码和可重试性。

## 已确认 API

- Google 使用 Gemini 官方 API，默认模型为 Nano Banana 2 `gemini-3.1-flash-image`。官方文档说明它支持文本和图片组合输入，并推荐该模型作为质量、成本和延迟的通用平衡：<https://ai.google.dev/gemini-api/docs/image-generation>。
- OpenAI 使用官方 Image API，模型为 `gpt-image-2`。文生图调用 `/v1/images/generations`，图生图调用 `/v1/images/edits`：<https://developers.openai.com/api/docs/guides/image-generation>。
- OpenAI `gpt-image-2` 的图像输入始终按高保真处理，不提供可调 `input_fidelity`；“保留原图/自由重绘”只作为提示词意图，不伪装为服务商数值参数。
- `gpt-image-2` 当前不支持透明背景，因此能力矩阵中必须禁用透明背景选项。

两家官方安全文档都要求生产 Web 应用不要把 API Key 暴露在客户端，故本模块固定使用轻量服务端代理，密钥从 `GEMINI_API_KEY` 和 `OPENAI_API_KEY` 环境变量读取。

## 失败场景

- API 名称相同但实际 Base URL、字段或鉴权协议不同。
- 浏览器跨域阻止直连，或密钥暴露在前端请求中。
- “测试连接”意外触发计费生成。
- 服务商返回成功状态但没有图片，或只返回文本说明。
- 图生图参考图已经释放、过期或上传失败。
- 同一按钮重复提交产生多次计费。
- 网络超时后直接重试，实际服务商已接受上一请求。
- 模型能力变化后继续展示已经不支持的参数。
- 提示词覆盖升级后无法还原历史生成输入。
- 图片历史保留远程临时 URL，刷新后全部失效。

## 安全和性能影响

- **密钥**：固定使用轻量代理和服务端环境变量；H5 不接收、不显示、不保存官方 API Key。
- **日志**：Authorization、API Key、签名 URL、原始图片字节和完整敏感错误体必须脱敏。
- **隐私**：提交图生图前明确告知参考图将发送到哪个服务商。
- **成本**：提交和重新生成是显式用户操作；连接测试不得隐式计费。
- **性能**：大图解码和预览避免阻塞 UI；历史列表使用缩略图，原图按需打开。
- **资源**：对象 URL 在不再使用时释放，但当前源图和未完成任务引用不能提前释放。

# 基础 H5 UI 框架

## 目标

- 建立可运行的 H5 应用外壳，为各生产力工具模块提供稳定页面入口。
- 使用左侧页签栏承载“生图、生成序列帧、序列帧调整（导出）、设置”四个一级页面。
- 先保证结构、状态和可操作性，不投入视觉美术与品牌设计。
- 让后续整体美术优化只替换样式和组件表现，不重写页面路由与业务状态。

## 非目标

- 不在本模块中完成最终视觉设计、动画、图标系统或品牌规范。
- 不以移动端窄屏为首期主要工作区；MVP 采用桌面浏览器优先。
- 不在本模块实现序列帧生成、帧编辑或导出业务。
- 不允许页面组件直接调用具体 AI 服务商 API。

## 判级理由

本变更会确定前端技术栈、应用入口、一级导航、页面生命周期和共享 UI 边界，是所有产品功能的运行基础，属于大模块。

## 影响模块

- `APP`：新增应用启动、路由、左侧页签栏、页面出口和全局错误区域。
- `SHARED`：新增无视觉绑定的基础面板、按钮、输入、状态和消息组件。

## 当前行为

- 仓库没有前端构建配置、依赖清单或产品代码。
- `src/app` 和 `src/shared` 只有职责说明。
- 已批准的产品模块没有可访问的页面入口。

## 目标行为

### 技术栈建议

- React + TypeScript：承载跨页状态、任务生命周期和后续帧编辑交互。
- Vite：提供轻量 H5 开发与构建入口。
- React Router：让页签对应稳定 URL，刷新和直接访问不会丢失当前页面。
- 原生 CSS 与少量设计变量：不引入重型 UI 组件库，避免锁定后续美术方案。

该技术栈属于 Draft 建议，必须随本文档一并批准后才能创建代码。

### 一级页面

| 页签 | 路由 | 首期状态 | 职责 |
|---|---|---|---|
| 生图 | `/image` | 首个实现模块 | 文生图、图生图、本地图片确认和生成历史 |
| 生成序列帧 | `/sequence` | 占位页面 | 源图、序列预设、用户描述和生成任务 |
| 序列帧调整（导出） | `/frames` | 占位页面 | 帧预览、排序、删除、局部重试和导出 |
| 设置 | `/settings` | 最小可用 | API 连接和基础提示词模板设置 |

- 默认路由为 `/image`。
- 未实现页面显示模块名称、规划状态和依赖条件，不使用空白屏幕。
- 页签只负责导航，不持有具体业务状态。

### 页面框架

```text
┌──────────────┬────────────────────────────────────────────┐
│ GIF CRAFT    │ 页面标题 / 全局任务或错误提示             │
│              ├────────────────────────────────────────────┤
│ 生图         │                                            │
│ 生成序列帧   │              当前页面内容                  │
│ 序列帧调整   │                                            │
│ 设置         │                                            │
│              │                                            │
└──────────────┴────────────────────────────────────────────┘
```

- 左侧页签栏固定显示，当前页签有明确选中状态。
- 主内容区使用简单面板、标题、表单和结果区域。
- 全局错误边界捕获页面异常并提供返回“生图”页的恢复入口。
- 全局通知只显示跨页面事件；表单错误留在字段或当前模块内。

## 接口与数据流

- `APP` 只装配页面、路由和全局依赖，不实现提示词或 API 请求。
- 各页面通过模块公开用例读取和修改业务状态。
- `SHARED` 组件只接收值、状态和回调，不导入业务模块。
- 页面卸载不能自动取消仍在服务商执行的计费任务；任务生命周期由对应模块管理。

基础 UI 状态至少统一表达：

- `idle`：可操作。
- `loading`：正在执行本地或远端操作。
- `success`：操作完成。
- `warning`：可继续，但存在风险或能力降级。
- `error`：操作失败并提供恢复动作。
- `disabled`：依赖条件不满足，并说明原因。

## 失败场景

- 页面刷新后路由丢失或进入空白页。
- 未实现页签点击后没有任何状态说明。
- 页面组件绕过用例直接访问服务商 API。
- 业务任务绑定页面生命周期，切换页签后任务失去跟踪。
- 全局错误导致整个页面无法恢复。
- 基础组件包含过多视觉假设，后续美术升级必须重写业务页面。

## 安全和性能影响

- **安全**：应用外壳不读取、保存或展示完整 API Key。
- **性能**：四个页面按路由拆分，为后续大型帧编辑模块预留懒加载能力。
- **兼容性**：首期支持现代桌面浏览器；具体浏览器版本在技术栈批准后写入长期文档。

## 实施步骤

1. 批准本文档并独立提交设计基线。
2. 初始化 React、TypeScript、Vite 和基础测试环境。
3. 创建应用启动入口、四个路由和左侧页签栏。
4. 创建简单共享面板、表单、按钮、状态和错误组件。
5. 为未实现模块创建明确占位页。
6. 接入生图模块和最小设置页面。
7. 执行路由、错误恢复、构建和基础可访问性测试。
8. 更新长期架构与模块索引并关闭本模块。

# 路线图

当前定位为内部生产工具；商业化和计费能力不在路线图中。

## M0：项目基础

- 明确产品目标、用户路径、首期范围和非目标。
- 定义分层架构、模块职责与 AI 适配边界。
- 建立只含职责说明的仓库目录。
- 确认技术栈、首个 AI 服务商和部署模式。

## M1：最小生成闭环

- [x] 完成文生图、图生图和本地图片直接上传三种源图入口。
- [x] 完成源图确认、替换和不可变任务引用。
- [x] 完成角色/场景类型，以及待机、攻击、其他角色动作预设。
- [x] 接入 Gorilla MCP，并声明文生图、图生图和 Seedance 图生序列能力。
- [x] 完成预设底词、用户描述和结构化参数合成，以及最终有效参数确认。
- [x] 完成任务提交、代理本地状态查询、错误提示、结果校验和稳定帧交接。
- [x] 支持序列帧基础预览、审核、移除/恢复、排序和不可变快照。
- [x] 支持基于不可变工作区快照、带 `manifest.json` 的 PNG ZIP 图片包导出。

## M2：可用性与可靠性

- [x] 支持能力驱动的异常帧局部候选重试、非破坏性移除和重新排序。
- [x] 加入序列任务刷新恢复、请求幂等、单活跃任务保护和轮询退避策略。
- [x] 完成本地源图库存、序列任务历史、来源追溯和工作区恢复入口。
- [ ] 完成本地项目草稿与项目级配置管理。
- 建立安全、性能和兼容性基线。

## M3：扩展能力

- 增加第二个 AI 服务商以验证适配层设计。
- 增加 GIF、WebP 或视频导出。
- 提供帧率、尺寸、裁切和质量等导出参数。
- 根据需求评估云端项目、团队协作和模板能力。

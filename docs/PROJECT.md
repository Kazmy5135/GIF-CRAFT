# 项目定义

## 产品愿景

让创作者在浏览器中用尽可能少的操作，将文字创意或参考素材转化为结构清晰、可预览、可调整、可导出的序列帧动画资产。

## 目标用户

- 需要快速制作小游戏、H5 活动或原型动画的独立开发者。
- 需要批量制作动态视觉素材的设计师和内容创作者。
- 希望验证动画概念、但没有完整动画制作工具链的产品团队。

## 核心用户路径

1. 从“新生成”进入任务引导，或从“库存”恢复已有静态图/序列资产。
2. 选择 AI 服务商并配置访问凭据。
3. 通过文生图、图生图获得源图，直接上传本地图片，或从图库选择已有图片。
4. 显式确认静态图，选择角色或场景序列帧类型。
5. 角色序列帧选择待机、攻击或其他动作预设；场景序列帧使用固定镜头的场景预设。
6. 填写用户描述并确认帧数、源帧率、循环方式、画布和对齐等有效参数。
7. 提交图生序列帧任务并观察进度；同一静态图可多次创建互不覆盖的序列 ID。
8. 在工作区检查连续性，抽帧、排序、重试指定帧，并按需要覆盖播放 FPS。
9. 预览当前工作区结果，生成不可变快照并导出带清单的 PNG ZIP。

## 成功标准

- 用户能够完成“确认静态图—配置—生成—帧工作区整理—不可变快照—PNG ZIP 导出”的当前闭环。
- 用户能够从三种源图入口进入同一套图生序列帧流程。
- 用户能够从图库复用静态图，从序列帧库恢复工作区，并从旧序列发起不覆盖历史记录的整序列重做。
- 角色和场景资产使用不同的工程预设，生成任务能够复现其预设版本和有效参数。
- 单帧失败不会迫使整个任务从头开始。
- 更换 AI 服务商时，核心领域和用户工作流无需重写。
- 错误信息能指出问题类型，并给出可执行的恢复方式。
- 用户可以明确知道凭据、素材和结果分别保存在哪里。

## 产品约束

- 当前产品是内部生产工具，本轮不建设价格展示、额度、预算、消费台账、支付、订阅或其他商业化/计费能力。
- H5 浏览器环境受跨域、内存、持久化容量和并发请求限制。
- 不同 AI 服务商在鉴权、异步任务、输出格式和限流策略上存在差异。
- 序列帧对角色、构图和运动连续性要求高，必须支持局部重试与结果筛选。
- 不同 AI 服务商对帧数、帧率和时长的支持不同，提交前必须展示映射后的最终有效参数。
- 大批量图像处理可能需要后端代理或异步任务服务，不能假设全部在浏览器内完成。

## 首期验收边界

已完成的序列生成 MVP 接通 Gorilla MCP Seedance 2.0 fast，将 1:1、480p、4 秒 H.264 视频在服务端校验并均匀抽取为 8 或 12 个 PNG 帧。角色预设包含待机、攻击和其他；场景提供通用固定镜头预设。任务具备提交前有效参数确认、幂等保护、刷新恢复、状态未知隔离、本地 Blob 持久化、完整性校验和帧工作区交接。

帧工作区已交付：按可覆盖的播放 FPS 和循环方式连续预览，逐帧审核与抽帧，非破坏性移除/恢复，稳定排序，能力驱动的指定帧重试，整序列重做入口，以及不可变工作区快照。当前 Seedance 没有原生单帧能力，明确降级为完整子任务重生成并只提取同一原始索引候选；候选必须由用户显式接受，且可恢复原版。播放 FPS 只改变工作区 revision 和后续快照，不改写原生成任务或旧快照。

资产库存与首期导出已交付：图库复用现有 `SourceImageAsset`，序列帧库按 `GenerationJob.id` 展示来源、状态和资源可用性；整序列重做使用新的任务 ID 和 `redoOfJobId`，旧序列、工作区与快照保持不变。PNG ZIP 导出只读取不可变快照，顺序输出 PNG 并附带 `manifest.json`；GIF、WebP、视频编码仍属于后续扩展。高级时间轴编辑、原视频重采样、专业图片编辑、团队协作、云端项目同步、模型训练和专业视频后期不属于首期验收范围。

# 系统架构

## 总体结构

系统按“界面与功能、核心领域、基础设施”分层。页面和组件只发起用例，不直接拼接服务商请求；外部 AI API 通过统一接口适配，任务状态由核心层管理。

```text
H5 界面
  │
  ├─ 项目配置 / 生成面板 / 帧预览 / 导出面板
  │
应用用例
  │
  ├─ 创建任务 / 查询进度 / 重试单帧 / 整理帧 / 导出
  │
核心领域
  │
  ├─ Project / GenerationJob / Frame / ExportJob
  │
基础设施
  ├─ AI Gateway → Provider Adapter → 外部 AI API
  ├─ Storage Gateway → 浏览器存储或未来的云端存储
  └─ Export Gateway → 图片包、GIF、WebP 或视频实现
```

## 关键模块

### `src/app`

负责应用启动、路由、全局错误边界、依赖装配和页面级生命周期。该层不承载具体 AI 服务商逻辑。

### `src/features`

按用户可感知能力组织功能，包括项目管理、生成任务、帧工作区和导出。功能模块通过核心用例协作，避免跨功能直接访问内部状态。

### `src/core`

包含与 UI 框架和服务商无关的领域对象、任务状态、业务规则和用例契约。它是稳定层，应能够脱离浏览器页面进行测试。

建议的生成任务状态：

```text
draft → queued → submitting → generating → processing → completed
                   │              │            │
                   └──────────────┴────────────┴→ failed → retrying
                                                        │
                                                        └→ cancelled
```

### `src/infrastructure`

实现外部系统接入：AI 服务商适配、受控代理、浏览器存储、文件处理和导出。服务商差异在此收敛，不向上层泄漏专有字段。

### `src/shared`

保存真正跨模块复用的 UI 基础、通用类型、错误表示和小型工具。业务规则不能为了“复用”而下沉到该目录。

## 建议部署边界

### 纯前端模式

适合允许浏览器安全直连、支持跨域且用户自行提供密钥的 AI 服务。优点是部署简单；风险是密钥暴露面、浏览器限制和任务可靠性较难控制。

### H5 + 轻量代理模式（推荐）

H5 负责交互和预览，受控代理负责隐藏服务端凭据、统一鉴权、限流、请求签名、回调接收和日志脱敏。是否保存用户素材必须有明确策略和用户提示。

## 数据边界

- `Project`：项目元数据、提示词、画面规格和帧序列引用。
- `GenerationJob`：服务商无关的任务参数、状态、进度、错误和重试信息。
- `Frame`：帧索引、来源任务、资源地址或本地引用、尺寸和审核状态。
- `ProviderConfig`：服务商类型、非敏感配置及凭据引用；不得将明文密钥写入仓库。
- `ExportJob`：导出格式、帧率、尺寸、质量和输出状态。

## 质量要求

- 所有任务状态变化可追踪、可解释。
- 网络超时、限流、鉴权失败和服务商错误需映射为统一错误类型。
- 重试应具备幂等策略，避免重复计费或产生不可识别的重复任务。
- 帧顺序使用稳定索引，删除或重试不得破坏原始序列关系。
- 对 API 密钥、用户提示词和素材地址执行分级日志与脱敏。

## 待决策项

- 前端技术栈与构建工具。
- 首个 AI 服务商及其序列一致性能力。
- 纯前端或轻量代理的最终部署模式。
- 首期导出格式优先级。
- 浏览器本地存储方案及容量上限。

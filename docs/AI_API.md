# AI API 接入约定

## 目标

用统一的服务商接口隔离外部 AI API 差异，使生成工作流不依赖特定厂商的鉴权方式、字段命名或任务状态。

## 统一能力边界

每个服务商适配器至少需要表达以下能力：

- 校验配置与鉴权信息。
- 声明是否支持文生图、图生图和图生序列帧。
- 提交文生图任务。
- 提交图生图任务。
- 使用已确认源图提交序列帧生成任务。
- 查询异步任务状态与进度。
- 获取生成结果及必要的元数据。
- 取消任务（服务商支持时）。
- 将服务商错误映射为统一错误类型。

源图准备阶段已实现配置状态、文生图和图生图。序列生成阶段已实现图生序列、代理本地任务查询和结果获取；当前 Seedance MCP 工具不提供外部任务 ID、服务商查询、取消或真实进度，因此这些能力被明确声明为不支持，不由页面伪造。

## 已实现的同源代理 API

H5 只调用同源代理：

| 方法 | 路径 | 作用 |
|---|---|---|
| `GET` | `/api/health` | 代理存活检查 |
| `GET` | `/api/providers` | 返回服务商配置状态和能力矩阵，不返回密钥 |
| `POST` | `/api/source-images/generate` | 提交统一文生图或图生图请求 |
| `POST` | `/api/sequence-jobs` | 校验并创建一次本地代理序列任务 |
| `GET` | `/api/sequence-jobs/:jobId` | 查询代理进程内的统一任务快照 |
| `GET` | `/api/sequence-jobs/:jobId/result` | 获取已校验并归一化的有序帧结果 |

代理端当前适配：

- Gorilla Canvas MCP：服务端通过官方 MCP TypeScript SDK v1 的兼容性 SSE Client 连接 `MCP_SERVER_URL`，Bearer 凭据由 `MCP_AUTH_TOKEN` 注入。`GET /api/mcp/tools` 只返回脱敏工具名称、说明和 `inputSchema`，用于完成工具与参数映射。
- Google Gemini 官方 API，默认模型 `gemini-3.1-flash-image`。
- OpenAI 官方 Image API，默认模型 `gpt-image-2`；文生图使用 `/v1/images/generations`，图生图使用 `/v1/images/edits`。
- 密钥分别由服务端环境变量 `GEMINI_API_KEY` 和 `OPENAI_API_KEY` 注入；模型可通过 `GEMINI_IMAGE_MODEL`、`OPENAI_IMAGE_MODEL` 覆盖。
- 同一个 `clientRequestId` 在单个代理进程内复用进行中或已完成结果，避免页面重复点击创建重复外部请求。
- 首期只向 H5 暴露 `request_failed`、`status_unknown` 和 `no_valid_image` 级别的脱敏状态，不展示精细厂商错误分类。

### MCP 图片工具约束

- 生图页默认选择 `Gorilla Banana`，并允许切换到 `Gorilla OpenAI Image2`；两者只有 URL 与轮换后的 Token 都配置后才允许提交。
- Banana 文生图、Banana 图生图和 Image2 工具名来自已验证的运行时 `tools/list`，工具不存在或 Schema 漂移时在调用前失败。
- 图生图先通过 `gorilla_upload_media` 上传参考图，并把返回的原始相对 `assetUrl` 传入模块。
- Gorilla 的 Image2 Schema 把 10 个说明为“可选”的图片槽位错误标成必填。适配器在文生图时上传并复用 256×256 空白占位图，在图生图时复用真实参考图 URL；该兼容层仅作用于 Image2。
- Banana 的质量映射为 `0.5K / 1K / 2K`；Image2 的质量映射为 `low / medium / high`，宽高比映射为其受支持尺寸。
- MCP 返回的相对 `/assets/` 路径只解析到已配置 MCP 同源 HTTPS 域名；允许的附加资源域名必须显式写入 `MCP_ASSET_HOSTS`。
- 代理下载结果并转为 PNG、JPEG 或 WebP data URL，限制重定向域名、响应格式和最大 24 MB，避免历史依赖短期 URL。
- MCP 调用超时进入 `status_unknown`，不会自动生成第二个外部请求。
- Banana 与 Image2 的文生图、图生图四条真实链路已通过轮换凭据验证。

### MCP Seedance 序列工具约束

- 首个序列提供方使用 `flowspace_module_1775547205956_bi14do`，模型为 Seedance 2.0；默认 `fast`，保留 `standard` 能力。
- 服务商输入固定映射为 1:1、480p、4 秒；四个 v1 预设仍以 8/12 帧和 8/12 FPS 表达最终帧数量及播放元数据，映射在提交前展示。
- MCP 调用同步返回 MP4 资源引用，没有可跨代理恢复的外部任务 ID。代理以随机 `proxyInstanceId` 绑定能力快照、请求、收据和查询；代理实例变化后旧任务进入 `status_unknown`，不能自动重提。
- 代理仅允许一个活跃序列任务，按 `clientRequestId` 做幂等和冲突检测，并保留受限结果缓存、任务元数据与 tombstone，防止短窗口重复提交。
- MP4 下载限制重定向、主机、端口、MIME、文件签名和 64 MiB 体积；`ffprobe` 校验 H.264、正方形和约 4 秒时长，`ffmpeg` 无 shell 启动并均匀输出精确 8/12 个 PNG 帧。
- `FFMPEG_PATH` 与 `FFPROBE_PATH` 可覆盖命令路径；未配置时使用运行环境 `PATH`。
- 客户端只在完整帧数量、稳定索引、任务归属、相同尺寸、允许 MIME 和 Blob 可读性全部通过后将任务标记为 `completed`。
- 真实验收已覆盖一个 8 帧角色任务、一个 12 帧场景任务、代理重启后的未知状态，以及非整秒视频精确抽帧。
- 序列能力通过 `frameRetryMode` 声明指定帧重试模式。当前 Gorilla Seedance 声明 `full_sequence_fallback`：重新生成完整序列，只把目标原始 `sequenceIndex` 的帧作为候选，不自动覆盖工作区当前帧。
- 工作区重试使用独立且已持久化的 `draftId`、`clientRequestId` 和尝试 ID。父任务业务参数保持冻结，但新的子任务使用当前能力快照的 `proxyInstanceId`；提交回执和子任务 ID 必须在首次查询前持久化。
- 运行中或未知的重试刷新后只按已持久化子任务 ID 查询，不重新 POST；没有可查询 ID 时允许用户显式放弃本地跟踪，并明确远端任务仍可能运行。

MCP SDK 的远程客户端使用和工具发现依据官方 [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk)；该服务使用旧式 HTTP+SSE，因此采用 v1 SDK 的 `SSEClientTransport` 兼容路径。

## 标准输入概念

- 提示词与负面提示词。
- 参考图或参考资源。
- 已确认源图资产及不可变资源引用。
- 序列帧类型：角色或场景。
- 角色动作预设：待机、攻击或其他。
- 预设 ID、预设版本和编译后的提示词快照。
- 帧数、帧率、循环方式、锚点、画布尺寸、宽高比和随机种子。
- 运动描述、风格约束和连续性参数。
- 服务商扩展参数；必须与核心字段隔离。

## 标准输出概念

- 内部任务 ID 与服务商任务 ID。
- 统一任务状态、进度和时间信息。
- 服务商实际采用的帧数、帧率、时长和其他有效参数。
- 按稳定索引排列的帧结果。
- 每帧的资源引用、尺寸、格式和生成元数据。
- 可恢复性标识、重试建议与统一错误信息。

## 安全约定

- 禁止把真实 API Key、Token、Cookie 或签名参数提交到 Git。
- 不在普通日志中记录完整鉴权头、原始密钥或包含敏感查询参数的 URL。
- 浏览器直连时，仅允许用户主动提供的密钥，并清楚说明保存位置和有效期。
- 代理模式中，服务端凭据通过部署环境的密钥管理注入。
- 外部资源地址可能短期失效，导出前需确认资源可读取或已安全缓存。
- 本地图片只有在用户明确提交对应 AI 操作后才能发送给外部服务。

## 统一能力接口

网关在领域层表达三种独立能力：

```text
generateImageFromText(request) → SourceImageJob
generateImageFromImage(request) → SourceImageJob
generateSequenceFromImage(request) → SequenceGenerationJob
```

本地图片直接上传不调用前两个接口。图片校验和确认完成后，将其规范化为 `SourceImageAsset`，仅在提交图生序列帧时调用第三个接口。

服务商适配器可以把统一参数转换为厂商字段，但必须返回最终有效参数。如果服务商不支持请求的帧数、帧率或时长，系统必须在外部请求提交前向用户展示映射结果。

## 序列预设与提示词

MVP 的提示词按以下顺序合成：

```text
公共游戏工程底词
  + 角色/场景类型底词
  + 待机/攻击/其他动作底词
  + 用户描述
  + 公共负向约束
```

- 公共底词和负向约束保证固定画布、稳定对齐、主体一致和有序时间推进。
- 角色预设使用底部中心锚点；场景预设使用固定镜头和完整画布。
- 用户描述补充动作、场景运动、情绪和节奏，不能覆盖硬工程约束。
- 帧数、帧率、循环方式、锚点和画布始终是结构化参数；提示词不是这些数据的唯一事实源。
- 预设内容必须版本化。任务保存预设 ID、版本、编译提示词和最终有效参数，保证历史请求可复现。
- 批准的首期预设和底词内容见 [`MOD-20260710-002`](../AIwork/2026-07-10/MOD-20260710-002-source-image-sequence-flow.md)。

## 可靠性约定

- 为任务提交生成客户端幂等标识。
- 序列任务提交时冻结源图引用、预设版本和有效参数，之后更换项目当前源图不影响历史任务。
- 为查询接口设置合理的退避、抖动和最大轮询时间。
- 区分可重试错误与永久错误，避免无效重试和重复外部任务。
- 对限流、超时、服务不可用和内容安全拒绝分别处理。
- 保留原始错误编号用于诊断，但向用户展示可理解的恢复建议。

## 服务商选型清单

- 是否支持参考图、角色一致性或关键帧约束。
- 是否分别支持文生图、图生图和图生序列帧，是否允许组合不同能力提供商。
- 是否支持真正的序列生成，或只能逐帧生成。
- 支持的帧数、帧率、时长、循环模式和对齐控制范围。
- 是否提供异步任务查询、回调和取消能力。
- 是否允许 H5 跨域直连；密钥暴露风险是否可接受。
- 并发限制、速率限制和结果保留时长。
- 输出格式、分辨率、内容安全政策和内部使用约束。

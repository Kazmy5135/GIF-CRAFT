# AI API 接入约定

## 目标

用统一的服务商接口隔离外部 AI API 差异，使生成工作流不依赖特定厂商的鉴权方式、字段命名或任务状态。

## 统一能力边界

每个服务商适配器至少需要表达以下能力：

- 校验配置与鉴权信息。
- 声明是否支持文生图、图生图和图生序列帧。
- 提交文生图任务。
- 提交图生图任务。
- 使用已确认源图提交序列帧生成任务。
- 查询异步任务状态与进度。
- 获取生成结果及必要的元数据。
- 取消任务（服务商支持时）。
- 将服务商错误映射为统一错误类型。

## 标准输入概念

- 提示词与负面提示词。
- 参考图或参考资源。
- 已确认源图资产及不可变资源引用。
- 序列帧类型：角色或场景。
- 角色动作预设：待机、攻击或其他。
- 预设 ID、预设版本和编译后的提示词快照。
- 帧数、帧率、循环方式、锚点、画布尺寸、宽高比和随机种子。
- 运动描述、风格约束和连续性参数。
- 服务商扩展参数；必须与核心字段隔离。

## 标准输出概念

- 内部任务 ID 与服务商任务 ID。
- 统一任务状态、进度和时间信息。
- 服务商实际采用的帧数、帧率、时长和其他有效参数。
- 按稳定索引排列的帧结果。
- 每帧的资源引用、尺寸、格式和生成元数据。
- 可恢复性标识、重试建议与统一错误信息。

## 安全约定

- 禁止把真实 API Key、Token、Cookie 或签名参数提交到 Git。
- 不在普通日志中记录完整鉴权头、原始密钥或包含敏感查询参数的 URL。
- 浏览器直连时，仅允许用户主动提供的密钥，并清楚说明保存位置和有效期。
- 代理模式中，服务端凭据通过部署环境的密钥管理注入。
- 外部资源地址可能短期失效，导出前需确认资源可读取或已安全缓存。
- 本地图片只有在用户明确提交对应 AI 操作后才能发送给外部服务。

## 统一能力接口

网关在领域层表达三种独立能力：

```text
generateImageFromText(request) → SourceImageJob
generateImageFromImage(request) → SourceImageJob
generateSequenceFromImage(request) → SequenceGenerationJob
```

本地图片直接上传不调用前两个接口。图片校验和确认完成后，将其规范化为 `SourceImageAsset`，仅在提交图生序列帧时调用第三个接口。

服务商适配器可以把统一参数转换为厂商字段，但必须返回最终有效参数。如果服务商不支持请求的帧数、帧率或时长，系统必须在计费请求提交前向用户展示映射结果。

## 序列预设与提示词

MVP 的提示词按以下顺序合成：

```text
公共游戏工程底词
  + 角色/场景类型底词
  + 待机/攻击/其他动作底词
  + 用户描述
  + 公共负向约束
```

- 公共底词和负向约束保证固定画布、稳定对齐、主体一致和有序时间推进。
- 角色预设使用底部中心锚点；场景预设使用固定镜头和完整画布。
- 用户描述补充动作、场景运动、情绪和节奏，不能覆盖硬工程约束。
- 帧数、帧率、循环方式、锚点和画布始终是结构化参数；提示词不是这些数据的唯一事实源。
- 预设内容必须版本化。任务保存预设 ID、版本、编译提示词和最终有效参数，保证历史请求可复现。
- 批准的首期预设和底词内容见 [`MOD-20260710-002`](../AIwork/2026-07-10/MOD-20260710-002-source-image-sequence-flow.md)。

## 可靠性约定

- 为任务提交生成客户端幂等标识。
- 序列任务提交时冻结源图引用、预设版本和有效参数，之后更换项目当前源图不影响历史任务。
- 为查询接口设置合理的退避、抖动和最大轮询时间。
- 区分可重试错误与永久错误，避免无效重试和额外计费。
- 对限流、超时、服务不可用和内容安全拒绝分别处理。
- 保留原始错误编号用于诊断，但向用户展示可理解的恢复建议。

## 服务商选型清单

- 是否支持参考图、角色一致性或关键帧约束。
- 是否分别支持文生图、图生图和图生序列帧，是否允许组合不同能力提供商。
- 是否支持真正的序列生成，或只能逐帧生成。
- 支持的帧数、帧率、时长、循环模式和对齐控制范围。
- 是否提供异步任务查询、回调和取消能力。
- 是否允许 H5 跨域直连；密钥暴露风险是否可接受。
- 计费方式、并发限制、速率限制和结果保留时长。
- 输出格式、分辨率、内容安全政策和商用授权条件。

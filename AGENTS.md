# AGENTS.md

本文件适用于整个仓库。所有 Agent 在分析、修改、审查或提交项目前都必须遵守本规范。

## 开始任务前

1. 阅读根目录 [`MODULE_INDEX.md`](MODULE_INDEX.md)。
2. 定位用户需求涉及的模块。
3. 阅读模块索引链接的长期文档和最近有效的大模块文档。
4. 只读取目标模块入口、直接依赖和相关测试；索引不足时才扩大搜索范围。
5. 如果模块索引与实际仓库不一致，把修正索引纳入当前任务。

不得为了“熟悉项目”而默认扫描全部代码。`docs/` 是长期事实，`AIwork/` 是设计与审计历史，二者不能互相替代。

## 强制变更分级

每次收到开发需求，Agent 必须先向用户输出：

```text
变更等级：大模块 / 小修改
判定依据：
影响模块：
文档动作：
```

代码行数和文件数量不能单独决定等级。边界不清时按大模块处理。

### 大模块

满足任意一项即为大模块：

- 新增顶层功能、完整用户流程或独立业务模块。
- 新增 AI 服务商、存储、导出、鉴权或代理能力。
- 改变模块职责、跨模块调用方向或核心依赖。
- 改变公开接口、领域模型、持久化结构或任务状态机。
- 涉及安全、计费、并发、兼容性或数据迁移。
- 无法确认影响范围或存在明显架构决策。

大模块必须执行：

1. 按香港时区在 `AIwork/YYYY-MM-DD/` 创建 `Draft` 主文档。
2. 使用 `MOD-YYYYMMDD-NNN-<ascii-slug>.md` 命名，并从 [`MODULE_SPEC`](AIwork/_templates/MODULE_SPEC.md) 模板开始。
3. 将受影响模块在 `MODULE_INDEX.md` 中标记为规划中并链接主文档。
4. 等待用户明确批准；未批准不得编写实现代码。
5. 批准后将状态改为 `Approved`，记录批准人和日期，并独立提交设计文档。
6. 实施时改为 `In Progress`。跨天工作在当天目录使用 [`MODULE_WORKLOG`](AIwork/_templates/MODULE_WORKLOG.md)，不得复制完整主文档。
7. 完成实现、验证、长期文档和模块索引同步后，依次更新为 `Implemented`、`Verified`、`Closed`。
8. 设计、实现、关闭必须形成可区分的 Git 提交。

### 小修改

只有同时满足以下条件才是小修改：

- 变更局限于一个已有模块。
- 不改变模块职责、公开接口、数据结构或状态模型。
- 属于局部逻辑修复、内部重构、交互细节或错误处理完善。
- 能通过现有测试或局部测试验证，并且容易回退。

小修改不需要事前文档，可以在判级后直接实施。用户发起 Code Review 时，再把尚未归档的小修改同步到当天 `AIwork/YYYY-MM-DD/CODE_REVIEW.md`。

## Code Review 归档

- 同一天的多次 Review 追加到同一个 `CODE_REVIEW.md`，使用 `CR-YYYYMMDD-NN` 分批编号。
- 默认范围从最近一次 Review 记录的 `head SHA` 到当前提交，并同时检查工作区。
- 第一次 Review 以最近一个已关闭大模块的实现基线为起点；如果不存在，则以项目治理建立后的提交为起点。
- 每条逻辑修改记录所属模块、行为变化、原因、风险、关键路径和验证结果。
- 同步记录 Review 发现、严重级别、处理状态和遗留问题。
- 未提交修改可以审查，但不能推进 Review 的 `head SHA`；它必须标记为 `Pending`，下次继续检查。
- 纯构建配置、格式化、编辑器配置、依赖锁文件和环境变量调整不进入逻辑修改清单。
- 如果配置实际改变鉴权、安全、运行逻辑、接口或持久化行为，必须重新判级，不能以“配置变更”为由排除。

详细规则和模板见 [`AIwork/README.md`](AIwork/README.md)。

## 模块索引维护

- `MODULE_INDEX.md` 只记录当前有效的模块职责、边界、入口、依赖和文档链接，不记录逐次小修改。
- 新增模块、改变职责、改变公开边界或改变依赖方向时必须更新索引。
- 局部逻辑调整只在每日 Code Review 中留档。
- 关闭大模块前必须确认索引与长期 `docs/` 已同步。

## 已关闭档案

- `Closed` 主文档不允许重写历史结论。
- 发现错误时只能追加带日期的“勘误”章节，或创建新的大模块文档并通过 `supersedes` 建立替代关系。
- 不得删除已被提交记录或其他文档引用的历史档案。

## Image Delivery Policy

- 在聊天中发送或引用图片时，先把图片保存到仓库根目录的 `codex-chat-images/` 临时目录。
- 验证文件是真实可读取的图片后，使用本地绝对路径嵌入，不能只依赖远程图片 URL。
- 文件名使用简短的 ASCII 字母、数字和连字符。

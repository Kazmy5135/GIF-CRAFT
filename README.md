# GIF CRAFT

GIF CRAFT 是一个面向浏览器的 H5 序列帧动画生产力工具。它通过接入外部 AI API，将用户的创意描述、参考素材和生成参数转换为连续序列帧，并提供预览、整理与导出工作流。

项目当前处于基础规划阶段：本仓库先明确产品目标、系统边界和模块职责，暂不包含业务代码。

## 项目目标

- 降低序列帧动画的制作门槛，让非专业动画用户也能快速完成从创意到可用素材的过程。
- 将外部 AI 能力封装为可替换的适配层，避免产品与单一模型或服务商强耦合。
- 提供稳定、清晰、可恢复的生成任务流程，包括参数配置、任务状态、失败重试和结果管理。
- 在 H5 环境中完成序列帧预览、顺序调整、基础筛选和导出。
- 优先保护用户密钥与素材，不在前端源码或持久化日志中暴露敏感凭据。

## 首期范围（MVP）

1. 创建序列帧生成任务：填写提示词、画面规格、帧数和生成参数。
2. 配置并调用一种外部 AI API。
3. 展示任务进度，并支持错误提示与安全重试。
4. 预览生成后的连续帧，支持删除、排序和重新生成异常帧。
5. 导出序列帧图片包，并为后续 GIF、WebP 或视频导出预留扩展能力。
6. 在浏览器本地保存非敏感项目配置和任务草稿。

## 非目标（当前阶段）

- 自研或训练图像生成模型。
- 提供完整的专业逐帧绘制、骨骼动画或视频剪辑能力。
- 在首期同时兼容大量 AI 服务商。
- 在未经用户明确授权的情况下将 API 密钥上传至第三方服务。

## 系统原则

- **服务商可替换**：所有 AI 请求通过统一网关和适配器进入外部服务。
- **任务驱动**：生成过程以任务状态机管理，避免页面刷新或网络波动导致状态不可理解。
- **浏览器优先**：交互、预览与轻量处理优先在 H5 客户端完成。
- **安全默认**：密钥最小暴露，日志默认脱敏，跨域调用优先通过受控代理。
- **渐进增强**：先形成最短生产闭环，再扩展更多服务商、编辑能力和导出格式。

## 基础目录

```text
.
├─ docs/               产品、架构、AI 接入与路线图文档
├─ public/             H5 静态资源占位
├─ src/
│  ├─ app/             应用装配、路由和全局生命周期
│  ├─ features/        按用户能力划分的业务功能
│  ├─ core/            与框架无关的领域模型和用例
│  ├─ infrastructure/  AI、存储、网络和导出等外部实现
│  └─ shared/          跨模块复用的 UI、类型和工具

```

目前各目录仅包含职责说明，不包含实现代码。详细设计见 [系统架构](docs/ARCHITECTURE.md) 和 [AI API 接入约定](docs/AI_API.md)。

## 里程碑

- M0：完成目标、架构、接口边界和仓库框架。
- M1：完成单一 AI 服务商的端到端生成闭环。
- M2：完成帧管理、局部重试与基础导出。
- M3：增加服务商适配、项目管理和更丰富的导出能力。

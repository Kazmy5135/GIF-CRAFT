# GIF CRAFT

GIF CRAFT 是一个面向浏览器的 H5 序列帧动画生产力工具。它通过接入外部 AI API，先帮助用户生成或确认一张源图，再结合游戏序列帧预设和用户描述生成连续帧，并提供预览、整理与导出工作流。

项目当前已完成基础 H5 应用外壳和源图准备模块。用户可以在“生图”页切换文生图、图生图或本地图片直通流程；序列帧生成与导出页面目前仍是明确的规划占位。

## 本地运行

要求 Node.js 24 或兼容当前依赖的现代 Node.js 版本。

```bash
npm install
copy .env.example .env
npm run dev
```

默认访问 `http://localhost:5173`。所有外部服务凭据只由服务端代理读取；不配置凭据时，本地图片直通和界面仍可使用，AI 生成按钮会保持禁用。

### Gorilla Canvas MCP

MCP 使用服务端 SSE 连接，真实令牌只放在本机 `.env`：

```env
MCP_SERVER_URL=https://canvas.dxx.cn/api/mcp/sse
MCP_AUTH_TOKEN=轮换后的新令牌
```

启动后，生图页默认提供“Gorilla Banana”和“Gorilla OpenAI Image2”，两者均支持文生图与图生图。设置页的“MCP 工具发现”可查看运行时工具名称和 Schema，不触发生图计费。此前发送到聊天中的令牌均视为已暴露，不能继续使用。

生产构建与启动：

```bash
npm run build
npm start
```

## 项目目标

- 降低序列帧动画的制作门槛，让非专业动画用户也能快速完成从创意到可用素材的过程。
- 将外部 AI 能力封装为可替换的适配层，避免产品与单一模型或服务商强耦合。
- 提供稳定、清晰、可恢复的生成任务流程，包括参数配置、任务状态、失败重试和结果管理。
- 在 H5 环境中完成序列帧预览、顺序调整、基础筛选和导出。
- 优先保护用户密钥与素材，不在前端源码或持久化日志中暴露敏感凭据。

## 首期范围（MVP）

1. 准备源图：支持文生图、图生图，也允许用户跳过 AI 源图生成并直接上传本地图片。
2. 配置序列帧：选择角色或场景类型；角色提供待机、攻击和其他动作预设。
3. 配置并调用一种外部 AI API，以已确认源图、版本化预设、用户描述和结构化参数生成序列帧。
4. 展示任务进度，并支持错误提示与安全重试。
5. 预览生成后的连续帧，支持删除、排序和重新生成异常帧。
6. 导出序列帧图片包，并为后续 GIF、WebP 或视频导出预留扩展能力。
7. 在浏览器本地保存非敏感项目配置、源图引用、预设版本和任务草稿。

## 非目标（当前阶段）

- 自研或训练图像生成模型。
- 提供完整的专业逐帧绘制、骨骼动画或视频剪辑能力。
- 在首期同时兼容大量 AI 服务商。
- 在未经用户明确授权的情况下将 API 密钥上传至第三方服务。

## 系统原则

- **服务商可替换**：所有 AI 请求通过统一网关和适配器进入外部服务。
- **任务驱动**：生成过程以任务状态机管理，避免页面刷新或网络波动导致状态不可理解。
- **游戏工程优先**：帧数、帧率、循环方式、画布和锚点使用结构化参数，提示词负责一致性和动作意图。
- **浏览器优先**：交互、预览与轻量处理优先在 H5 客户端完成。
- **安全默认**：密钥最小暴露，日志默认脱敏，跨域调用优先通过受控代理。
- **渐进增强**：先形成最短生产闭环，再扩展更多服务商、编辑能力和导出格式。

## 基础目录

```text
.
├─ AGENTS.md          Agent 工作规范与强制变更分级
├─ MODULE_INDEX.md    当前模块职责、边界、依赖与阅读入口
├─ AIwork/            按日期保存大模块、工作记录和 Code Review

├─ server/             轻量代理与 Gorilla MCP、Gemini、OpenAI 适配器
├─ src/
```

## 文档治理

本项目使用严格文档控制，不采用无边界的 Vibe Coding：

- 所有开发需求先由 Agent 判定为“大模块”或“小修改”。
- 大模块必须先在 `AIwork/` 建立设计文档并取得用户批准，之后才能实施。
- 小修改可以直接实施，在用户主动发起 Code Review 时归档当天的逻辑变化。
- 新 Agent 应先阅读 [`AGENTS.md`](AGENTS.md) 和 [`MODULE_INDEX.md`](MODULE_INDEX.md)，再按索引读取目标模块，避免扫描全部代码。
- 完整规则和模板见 [`AIwork/README.md`](AIwork/README.md)。

## 里程碑

- M0：完成目标、架构、接口边界和仓库框架。
- M1：完成源图准备、角色/场景预设和单一 AI 服务商的端到端序列帧生成闭环。
- M2：完成帧管理、局部重试与基础导出。
- M3：增加服务商适配、项目管理和更丰富的导出能力。
